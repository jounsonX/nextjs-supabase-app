## 1. 고객 세그먼트 (Customer Segments)

### 주 고객

| 세그먼트                 | 설명                                                                      | 규모 추정              |
| ------------------------ | ------------------------------------------------------------------------- | ---------------------- |
| **모임 주최자 (Host)**   | 수영·헬스·친구 모임 등 5~20인 소규모 친목 모임을 정기적으로 운영하는 개인 | 초기 타깃              |
| **모임 참여자 (Member)** | 주최자가 개설한 모임에 참여하는 구성원                                    | Host 1명당 평균 5~19명 |

### 보조 고객

| 세그먼트                  | 설명                                                               |
| ------------------------- | ------------------------------------------------------------------ |
| **플랫폼 운영자 (Admin)** | 서비스 전체를 관리하는 운영자 (사용자 역할 관리, 부적절 모임 통제) |

### 얼리 어답터

- 카카오톡 단체방에서 모임 운영의 번거로움을 경험한 소모임 주최자
- 수영·헬스 등 정기 모임을 운영하며 정산·카풀 조율에 피로를 느끼는 사람

## 2. 문제 (Problem)

### 핵심 문제 3가지

| 순위   | 문제                                                                                      | 현재 대안(고통 수준)                   |
| ------ | ----------------------------------------------------------------------------------------- | -------------------------------------- |
| **#1** | **정보 분산**: 공지·참여 여부·정산·카풀 정보가 카카오톡 단체방 채팅 속에 뒤섞여 관리 불가 | 카카오톡 단체방 채팅 (★★★★★)           |
| **#2** | **정산 번거로움**: 참여자 수 변동 시 비용 재계산과 개별 확인 메시지 전송 반복             | 엑셀/메모장 수기 계산 + 개별 DM (★★★★) |
| **#3** | **카풀 조율 혼란**: 탑승 가능 인원·출발지 확인을 채팅 스크롤로 찾아야 함                  | 채팅 핀 메시지 또는 별도 폼 (★★★)      |

### 기존 대안의 한계

- **카카오톡 단체방**: 공지가 채팅에 묻힘, 참여 현황 파악 불가, 정산 기능 없음
- **네이버 카페/밴드**: 모임 특화 정산·카풀 기능 없음, UX 복잡
- **스프레드시트 공유**: 실시간성 부족, 모바일 입력 불편, 별도 공지 채널 필요

## 4. 해결책 (Solution)

### 핵심 기능 → 문제 매핑

| 문제           | 해결책                                         | 핵심 기능 |
| -------------- | ---------------------------------------------- | --------- |
| 정보 분산      | 모임 단위 허브 페이지 (정보/공지/정산/카풀 탭) | F001~F006 |
| 정산 번거로움  | 1/n 자동 계산(올림) + 참여자별 완료 체크       | F009      |
| 카풀 조율 혼란 | 드라이버 등록 + 선착순 동승 신청 자동 처리     | F010      |

### MVP 범위 내 해결책

- **모임 생성/수정/삭제**: 제목·날짜·장소·정원·비용 한 번에 입력
- **참여 신청/승인/대기**: 정원 초과 시 자동 waitlisted 처리 후 빈자리 발생 시 승급
- **공지 관리**: 핀 고정 공지 최상단 표시, Host 전용 작성·삭제
- **정산 자동화**: `Math.ceil(비용 / 승인 참여자 수)` 계산, 완료 체크
- **카풀 매칭**: 출발지·좌석 수 등록 → 선착순 자동 수락

## 5. 고유 가치 제안 (Unique Value Proposition)

### 메인 UVP

> **"소모임 운영의 모든 것을 한 곳에 — 카카오톡 없이 공지·참여·정산·카풀을 5분 안에 해결"**

### 보조 UVP (세그먼트별)

| 세그먼트   | 가치 제안                                                                                           |
| ---------- | --------------------------------------------------------------------------------------------------- |
| **Host**   | 모임 생성부터 정산 완료까지 하나의 앱에서 처리. 카카오톡 채팅 스크롤 없이 참여자 현황을 한눈에 파악 |
| **Member** | 참여 신청 상태 실시간 확인, 1인당 부담 금액 즉시 조회, 카풀 자리 선착순 신청                        |

### 핵심 차별점

- **모임 특화**: 범용 협업 툴이 아닌 5~20인 친목 모임에 최적화된 UX
- **카카오 로그인**: 이미 익숙한 소셜 로그인으로 진입 장벽 최소화
- **모바일 우선**: 하단 탭 + FAB 기반 모바일 네이티브 수준 UX

## 3. 핵심 지표 (Key Metrics)

### 성장 지표

| 지표                      | 설명                            | 목표 (MVP 출시 3개월) |
| ------------------------- | ------------------------------- | --------------------- |
| **활성 모임 수**          | 최근 30일 내 open 상태 모임 수  | 50개                  |
| **주간 활성 사용자(WAU)** | 주 1회 이상 앱 접속 사용자      | 200명                 |
| **모임당 평균 참여자 수** | 모임 1개의 approved 참여자 평균 | 8명                   |

### 핵심 행동 지표

| 지표              | 설명                                     | 목표 |
| ----------------- | ---------------------------------------- | ---- |
| **정산 완료율**   | 모임 종료 후 payment_done 체크 완료 비율 | 70%  |
| **카풀 등록률**   | 모임에서 카풀 탭 이용 모임 비율          | 30%  |
| **Host 재사용률** | 모임을 2개 이상 생성한 Host 비율         | 40%  |

### 서비스 건강 지표

| 지표                     | 설명                                          |
| ------------------------ | --------------------------------------------- |
| **대기자 승급 처리율**   | waitlisted → approved 자동 전환 성공 비율     |
| **Admin 강제 처리 건수** | 부적절 모임 신고/강제 삭제 수 (낮을수록 건강) |

## 8. 채널 (Channels)

### 획득 채널

| 채널              | 방식                                            | 예상 효과                         |
| ----------------- | ----------------------------------------------- | --------------------------------- |
| **카카오 공유**   | Host가 모임 링크를 카카오톡으로 참여자에게 공유 | 바이럴 최적 — 기존 채팅 문화 활용 |
| **구글 검색**     | "소모임 관리 앱", "수영 모임 정산" 등 SEO       | 중장기 유기 유입                  |
| **커뮤니티 홍보** | 수영·헬스 관련 카페/오픈채팅에 직접 소개        | 얼리 어답터 확보                  |

### 활성화/유지 채널

| 채널                      | 방식                                       |
| ------------------------- | ------------------------------------------ |
| **이메일 인증**           | 회원가입 시 이메일 인증으로 신뢰성 확보    |
| **앱 내 알림** (MVP 이후) | Supabase Realtime 기반 참여 승인/공지 알림 |

### MVP 이후 수익 모델 후보

| 모델                        | 설명                                                | 우선순위 |
| --------------------------- | --------------------------------------------------- | -------- |
| **Freemium (모임 수 제한)** | 무료: 모임 3개 / 월, 유료: 무제한 + 고급 기능       | ★★★      |
| **정산 연동 수수료**        | 카카오페이·토스 정산 연동 시 건당 수수료 (MVP 이후) | ★★       |

## 9. 비용 구조 (Cost Structure)

### 고정 비용

| 항목         | 상세                                                | 월 예상 비용 |
| ------------ | --------------------------------------------------- | ------------ |
| **Supabase** | PostgreSQL + Auth + 스토리지 (Free → Pro 전환 기준) | $0 → $25/월  |
| **Vercel**   | Next.js 배포 (Hobby → Pro)                          | $0 → $20/월  |
| **도메인**   | 커스텀 도메인 유지비                                | ~₩15,000/년  |

### 변동 비용

| 항목                   | 상세                                      |
| ---------------------- | ----------------------------------------- |
| **Supabase 초과 사용** | MAU 50,000 초과, DB 8GB 초과 시 추가 과금 |
| **카카오 OAuth**       | 무료 (카카오 개발자 정책 기준)            |
| **Google OAuth**       | 무료 (월 10,000 인증 이하)                |

### 비용 구조 특성

- **초기 비용 최소화**: Supabase + Vercel 무료 티어로 MVP 출시 가능
- **사용량 기반 확장**: 사용자 증가에 따라 선형적으로 비용 증가
- **개발 비용**: 1인 개발자 기준 MVP 개발 약 6~8주

## 7. 불공정 우위 (Unfair Advantage)

> 단기적으로는 우위가 약하나, 아래 요소를 통해 경쟁 방어선 구축 예정

| 우위 요소              | 설명                                                                       |
| ---------------------- | -------------------------------------------------------------------------- |
| **카카오 사용자 락인** | 카카오 로그인 + 카카오 공유 연동으로 기존 모임 커뮤니티 자연 유입          |
| **모임 데이터 축적**   | 사용자 모임 이력·정산 데이터가 쌓일수록 이탈 비용 증가                     |
| **특화 UX**            | 5~20인 친목 모임에 최적화된 탭 구조 + 자동화 로직 (범용 툴 대비 진입 장벽) |
| **빠른 출시**          | Next.js + Supabase BaaS로 최소 인프라로 빠른 출시 → 선점 효과              |

## 가정 및 검증 계획

### 핵심 가정 (리스크 순)

| 순위   | 가정                                                         | 검증 방법                                               | 검증 시점         |
| ------ | ------------------------------------------------------------ | ------------------------------------------------------- | ----------------- |
| **#1** | Host가 카카오톡을 대체할 전용 툴로 MeetUp Manager를 채택한다 | 실제 모임 주최자 10명 대상 인터뷰 + 첫 모임 생성률 측정 | MVP 출시 후 2주   |
| **#2** | 참여자가 앱을 통해 정산 완료 체크를 실제로 사용한다          | 정산 완료율 지표 모니터링 (목표: 70%)                   | MVP 출시 후 1개월 |
| **#3** | 카풀 기능이 모임 참여 결정에 실질적 영향을 준다              | 카풀 등록률 + 동승 신청률 추적                          | MVP 출시 후 1개월 |
| **#4** | 이메일/소셜 로그인 진입 장벽이 낮아 가입 전환율이 높다       | 로그인 페이지 → 가입 완료 전환율 측정                   | MVP 출시 직후     |
