## 세부 설명

### 1. 문제 (Problem)

| 순위 | 문제                             | 현재 해결 방식       | 불편함 수준 |
| ---- | -------------------------------- | -------------------- | ----------- |
| 1    | 공지·참여관리·정산·카풀이 분산됨 | 카카오톡 단체방      | 높음        |
| 2    | 정산 금액 수동 계산 + 개별 요청  | 엑셀·카카오페이      | 높음        |
| 3    | 카풀 조율 텍스트 혼선            | 단체방 + 개별 메시지 | 중간        |
| 4    | 참여자 명단 확인 불편            | 댓글·이모지 반응     | 중간        |

### 2. 고객 세그먼트 (Customer Segments)

**주요 고객:**

- 수영/헬스 정기 운동 모임 주최자 (5~15명, 주 1~2회 반복)
- 소규모 동호회 운영자 (독서, 등산, 보드게임 등)
- 친구/지인 소셜 모임 주최자 (불정기, 10명 내외)

**얼리 어답터:**

- 직접 서비스를 사용할 개발자 본인 (수영 모임 운영 경험)
- 디지털 친화적인 20~40대 모임 주최자

### 3. 고유 가치 제안 (Unique Value Proposition)

> **"카카오톡 단체방 없이도, 소규모 모임을 주최자 혼자 체계적으로 운영할 수 있는 올인원 플랫폼"**

- 공지 → 참여 관리 → 정산 → 카풀을 하나의 화면에서 완결
- 별도 앱 설치 없이 웹 브라우저에서 바로 사용
- N/1 정산 자동 계산으로 "누가 얼마 냈는지" 확인 즉시 가능

### 4. 솔루션 (Solution)

| 기능                      | 해결하는 문제                 |
| ------------------------- | ----------------------------- |
| 공지 게시판 (핀 고정)     | 중요 공지 유실 방지           |
| 참여 신청/승인 시스템     | 참여자 명단 체계적 관리       |
| N/1 자동 정산 + 완료 체크 | 수동 계산 및 개별 독촉 제거   |
| 카풀 등록/선착순 매칭     | 카풀 조율 혼선 제거           |
| 탭 기반 모임 상세 허브    | 기능 분산 없이 한 페이지 집중 |

### 5. 채널 (Channels)

- **인지**: 구글 검색 SEO, 커뮤니티 바이럴 (클리앙, 운동 커뮤니티)
- **획득**: 주최자가 참여자에게 링크 공유 (자연스러운 바이럴)
- **활성화**: 첫 모임 생성 후 참여자 초대 경험
- **유지**: 정기 모임 반복 사용

### 6. 핵심 지표 (Key Metrics)

**검증 단계 (MVP):**

- 주간 활성 모임 수 ≥ 1 (개발자 본인 사용)
- 참여 신청 완료율 ≥ 80%
- 정산 완료율 ≥ 70%

**성장 단계:**

- 월간 신규 주최자 수
- 주최자 이탈률 (3개월 내 재사용 여부)
- 모임당 평균 참여자 수

### 7. 불공정 우위 (Unfair Advantage)

- **목적 특화 UX**: 범용 메신저(카카오톡) 대비 모임 운영에 최적화된 탭 구조
- **솔로 개발 속도**: 대형 서비스가 다루기 너무 작은 틈새 시장을 빠르게 점유
- **직접 사용자 관점**: 개발자 본인이 실제 수영 모임을 운영하며 페인포인트 체감

### 8. 비용 구조 (Cost Structure)

| 항목      | MVP 단계         | 성장 단계    |
| --------- | ---------------- | ------------ |
| Supabase  | 무료 (Free Tier) | $25/월 (Pro) |
| Vercel    | 무료 (Hobby)     | $20/월 (Pro) |
| 도메인    | —                | ~2만원/년    |
| 개발 비용 | 기회비용만       | —            |

**MVP 총 비용: 거의 $0**

### 9. 수익 구조 (Revenue Streams)

**MVP 단계**: 무료 운영 (사용자 검증 우선)

**향후 수익화:**
| 모델 | 내용 | 예상 단가 |
|------|------|---------|
| 프리미엄 구독 | 실시간 알림, 반복 일정, 초대 링크 | 월 2,900원 |
| 결제 연동 수수료 | 카카오페이/토스 정산 연동 | 건당 0.5~1% |
| B2B 플랜 | 기업 사내 동호회 관리 | 월 9,900원/팀 |

## 핵심 가정 및 검증 계획

| 가정                                                | 검증 방법                                 | 기준                           |
| --------------------------------------------------- | ----------------------------------------- | ------------------------------ |
| 주최자가 카카오톡 대신 전용 툴을 사용할 의향이 있다 | 직접 본인 모임에 사용 후 참여자 반응 수집 | 3명 이상 자발적 재사용         |
| 정산 자동화가 가장 큰 pain point다                  | 정산 탭 사용 빈도 측정                    | 모임당 정산 탭 클릭 ≥ 2회      |
| 카풀 기능이 참여율에 영향을 준다                    | 카풀 등록 모임 vs 미등록 모임 참여율 비교 | 등록 모임 참여율 10% 이상 높음 |

## MVP 범위 요약

**포함:**

- 모임 CRUD + 공지 + 참여 관리 + 정산 + 카풀 (웹, 로그인 필수)

**제외 (검증 후 추가):**

- 실시간 알림, 결제 연동, 초대 링크, 반복 일정, 모바일 앱
